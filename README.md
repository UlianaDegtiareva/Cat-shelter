# 🐾 Система управления приютом для кошек (Cat Shelter API)

## Постановка задачи №1: Архитектура справочной системы

**Цель:** Спроектировать и реализовать базовый REST API для первичного учета животных в приюте. Создать фундамент системы, обеспечивающий хранение структурированных данных и их корректную валидацию.

### Функциональные требования:

* **Сущность Cat:** Реализовать хранение данных о кошке со следующими полями:
    * `id` (автоматическая генерация)
    * `name` (имя/кличка)
    * `age` (полных лет)
    * `breed` (порода)
    * `history` (история появления в приюте)
    * `description` (описание характера/особенностей)
* **API Методы (CRUD):**
    * `POST /cats` — регистрация нового животного.
    * `GET /cats` — получение полного списка животных в приюте.
    * `GET /cats/:id` — получение детальной информации о конкретной кошке.
    * `DELETE /cats/:id` — исключение записи из базы данных.

### Технические требования и валидация:

1.  **Валидация входящих данных:**
    * Имя (`name`) не должно быть короче 2 символов.
    * Возраст (`age`) должен быть положительным целым числом.
2.  **Документация:** Полная поддержка Swagger UI с описанием всех полей и статус-кодов ответов на английском языке.
3.  **База данных:** Автоматическая синхронизация схемы данных с PostgreSQL через TypeORM.

**Ожидаемый результат:** Работающий Swagger-интерфейс, позволяющий наполнить базу первичными данными и протестировать корректность работы ограничений валидации.

## 🛠 Технологический стек

* **Runtime:** Node.js (v18+)
* **Framework:** NestJS
* **Database:** PostgreSQL (в Docker-контейнере)
* **ORM:** TypeORM
* **API Documentation:** Swagger UI
* **Validation:** Class-validator

## Инструкция по запуску

### 1. Первый запуск (Настройка)

Выполняется один раз при установке проекта.

1.  **Установка зависимостей:**
    ```bash
    npm install
    ```
2.  **Запуск базы данных (Docker):**
    Убедитесь, что Docker Desktop запущен, затем выполните:
    ```bash
    docker-compose up -d
    ```
    *Система создаст контейнер с PostgreSQL на порту `5433`.*

3.  **Запуск приложения:**
    ```bash
    npm run start:dev
    ```
    При возникновении ошибок на данном шаге типа:
    `EADDRINUSE: address already in use :::3000` необходимо сменить порт в самом коде или остановить процессы, которые занимают данный порт, так как ошибка означает, что порт 3000 уже занят. Освободить порт можно при помощи команды:
    ```bash
    Stop-Process -Id (Get-NetTCPConnection -LocalPort 3000).OwningProcess -Force
    ```
    После этого снова выполните запуск приложения `npm run start:dev`

### 2. Последующие запуски

Для повседневной работы достаточно убедиться, что контейнер БД запущен, и запустить сервер:
1. `docker-compose start` (если контейнер был остановлен)
2. `npm run start:dev`

Приложение будет доступно по адресу: [http://localhost:3000/api](http://localhost:3000/api)
