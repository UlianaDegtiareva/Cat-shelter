# 🐾 Система управления приютом для кошек (Cat Shelter API)

Информационная система для учета животных, волонтеров и процесса усыновления. Проект разработан на базе фреймворка **NestJS** с использованием **PostgreSQL**.

## 🛠 Технологический стек

* **Runtime:** Node.js (v18+)
* **Framework:** NestJS
* **Database:** PostgreSQL (в Docker-контейнере)
* **ORM:** TypeORM
* **API Documentation:** Swagger UI
* **Validation:** Class-validator

## Инструкция по запуску

### 1. Первый запуск (Настройка)

Выполняется один раз при установке проекта.

1.  **Установка зависимостей:**
    ```bash
    npm install
    ```
2.  **Запуск базы данных (Docker):**
    Убедитесь, что Docker Desktop запущен, затем выполните:
    ```bash
    docker-compose up -d
    ```
    *Система создаст контейнер с PostgreSQL на порту `5433`.*

3.  **Запуск приложения:**
    ```bash
    npm run start:dev
    ```
    При возникновении ошибок на данном шаге типа:
    `EADDRINUSE: address already in use :::3000` необходимо сменить порт в самом коде или остановить процессы, которые занимают данный порт, так как ошибка означает, что порт 3000 уже занят. Освободить порт можно при помощи команды:
    ```bash
    Stop-Process -Id (Get-NetTCPConnection -LocalPort 3000).OwningProcess -Force
    ```
    После этого снова выполните запуск приложения `npm run start:dev`

### 2. Последующие запуски

Для повседневной работы достаточно убедиться, что контейнер БД запущен, и запустить сервер:
1. `docker-compose start` (если контейнер был остановлен)
2. `npm run start:dev`

Приложение будет доступно по адресу: [http://localhost:3000/api](http://localhost:3000/api)
